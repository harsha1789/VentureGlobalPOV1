# HxGN SDx Document Control Bot — Technical Walkthrough

This system automates document control review for VG CP2 LNG submittals. There are 3 components that work together.

---

## 1. Mimic App (Mock HxGN SDx)

This simulates the real HxGN SDx application. It's a simple Express server + static HTML frontend.

- **Backend** (`hxgn-sdx-bot/mimic-app/backend/server.js`) — Express REST API on port 3000. Holds all submittal data in memory (seeded from `seed.js` on startup). Provides endpoints for login, todo list, claim, datasheet, document viewer, transmittal, approve, and reset.
- **Frontend** (`hxgn-sdx-bot/mimic-app/frontend/`) — Static HTML/CSS/JS that looks like the SDx interface. Has screens for login, work queue (todo list), submittal detail, load datasheet, document viewer, transmittal creation, approval, and final status.
- **Test PDFs** (`backend/test-pdfs/`) — 5 real PDF files generated by `pdfkit`. The server reads these using `pdf-parse` (Mozilla's PDF.js) to extract actual text content from the PDFs — this is not hardcoded or faked.

**Start it:**

```bash
cd hxgn-sdx-bot/mimic-app/backend
npm start
```

Runs on http://localhost:3000

**Login:** `dc_bot` / `BotPass2026!`

---

## 2. Bot (Playwright Automation)

The bot (`hxgn-sdx-bot/bot/bot.js`) is a headless Chromium browser controlled by Playwright. It mimics what a human document controller would do in SDx.

### What it does on startup

1. Resets the mimic app data (`POST /api/reset`)
2. Logs in as `dc_bot`
3. Scans the todo list and collects ALL rows with status "Submitted"
4. Loops through each document and runs the workflow below

### Per-document workflow

- **Claim** — Clicks the Claim button on the todo row. Server sets status to "In Review".
- **Validate Detail** — Reads the detail screen fields (doc number, revision, file type, originator, date) and checks they're populated.
- **Open & Validate Datasheet** — Opens the load datasheet, reads 5 fields (doc number, title, revision, file type, discipline), checks each passes validation.
- **Document Integrity & Quality** — Opens doc viewer, reads 3 basic checks from UI (format, corruption, size). Then calls the API directly (`/api/document/:id` and `/api/cross-validate/:id`) to run the real checks:
  - **12 document quality checks** — OCR/text searchability, draft watermarks, markups/annotations, page orientation, metadata on each page, security classification, revision history, document structure, contract reference, rev/issue purpose alignment, file completeness, second cover page.
  - **7 loadsheet cross-validation checks** — Compares what's extracted from the PDF against the loadsheet: document number, title, revision, discipline, file type, security class, issue purpose.
  - **Transmittal readiness** — Checks all 8 required fields can be extracted from the PDF before creating a transmittal.
- **Create Transmittal** — Generates a TRN number.
- **Approve** — Enters revision number, clicks approve.
- **Bot Reviewed** — Sets final status to "Bot Reviewed" (does NOT click Complete — that's reserved for manual DC review).

**If a document fails any quality/cross-validation check**, the bot logs the failure with details, takes a screenshot, and moves to the next document.

After all documents are processed, it generates an **HTML report** in `bot/reports/` with:

- Document summary table (pass/fail per document)
- Collapsible per-document sections showing every quality check, loadsheet comparison, and transmittal readiness check
- Workflow step log with screenshot links

**Run it:**

```bash
cd hxgn-sdx-bot/bot
HEADLESS=false node bot.js
```

Use `HEADLESS=false` to watch the browser.

## 3. How the PDF Validation Actually Works

This is the key part — the validation is real, not mocked.

When the bot hits the document integrity step, the server:

1. Reads the actual PDF file from `test-pdfs/` directory.
2. Uses `pdf-parse` (wrapper around Mozilla's PDF.js) to extract the text layer from the PDF binary.
3. Runs regex-based checks against the extracted text.
4. If the PDF has no text layer (like a scanned image), pdf-parse fails to extract anything and the OCR check fails.

### The 5 test documents

| Document | Type | What Happens |
|----------|------|-------------|
| `VG-CP2-MEC-DWG-0001_RevA.pdf` | Valid, complete | All metadata present, all checks pass. Bot Reviewed. |
| `VG-CP2-ELE-SPC-0042_RevB.pdf` | Valid, Rev B | Full revision history, all checks pass. Bot Reviewed. |
| `VG-CP2-PIP-DWG-0112_RevA.pdf` | Image-only | PDF has only vector graphics, zero text. pdf-parse returns empty string. **OCR FAIL: "image-only or scanned document"**. |
| `VG-CP2-CIV-CAL-0037_RevA.pdf` | Has markups | Real `[COMMENT]`, `[MARKUP]`, `/Annots` text in the PDF. **Markup detection FAIL**. |
| `VG-CP2-PRO-PID-0044_Rev0.pdf` | Multiple issues | Real "DRAFT" text in PDF, references "Cameron LNG" (wrong project), Rev 0 + AFC (misaligned), no security class, no revision history. **13 failures**. |

## API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/login` | POST | Authenticate user (dc_bot, admin) |
| `/api/todo-list` | GET | Return all submittals for the work queue |
| `/api/claim/:id` | POST | Claim a submittal, set status to "In Review" |
| `/api/submittal/:id` | GET | Return submittal detail fields |
| `/api/datasheet/:id` | GET | Return loadsheet row (5 fields) |
| `/api/document/:id` | GET | Read real PDF via pdf-parse, run 12 quality checks, return results |
| `/api/cross-validate/:id` | GET | Extract metadata from PDF, compare vs loadsheet and transmittal |
| `/api/transmittal/create` | POST | Generate TRN number for a submittal |
| `/api/submittal/approve` | POST | Mark submittal as approved with revision |
| `/api/submittal/bot-reviewed` | POST | Set final status to "Bot Reviewed" |
| `/api/reset` | POST | Re-seed all data to initial state |

## Quick Start

```bash
# Terminal 1 — Start the mimic app
cd hxgn-sdx-bot/mimic-app/backend
npm install
node generate-test-pdfs.js      # creates the 5 test PDFs (one-time)
npm start                        # server on :3000

# Terminal 2 — Run the bot
cd hxgn-sdx-bot/bot
npm install
npx playwright install chromium   # one-time
HEADLESS=false node bot.js        # runs against all 5 documents

# Report will be at: bot/reports/report-*.html
```

## Validation Checks Summary

| Category | Count | Source | Method |
|----------|-------|--------|--------|
| Basic integrity | 3 | UI (doc viewer) | File format, corruption, size |
| Document quality | 12 | API (`/api/document`) | pdf-parse text + regex: OCR, draft watermark, markups, orientation, structure, security class, revision history, contract ref, rev/issue alignment, completeness, second cover page, metadata per page |
| Loadsheet cross-validation | 7 | API (`/api/cross-validate`) | pdf-parse text + regex extraction, compared against datasheet fields |
| Transmittal readiness | 1-9 | API (`/api/cross-validate`) | Check all 8 required transmittal fields are extractable from PDF |
