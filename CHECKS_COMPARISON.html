Part 2: AI Validation Pipeline Checks
Tier 0: Pre-Flight Checks (T0)
# Check Name Method Pass Criteria
1 File Type Validation Regex: \.(pdf|doc|docx|xls|xlsx|dwg|txt)$ Extension in allowed list
2 File Not Empty Length check: text.length > 0 File has content, ≥ 20 chars
3 Filename Validity Trim/spaces check No leading/trailing/double spaces
4 Special Characters Regex: [~\/@&%!#$^*()...] No prohibited characters
5 Project Prefix Regex: ^(C2|CP2|VG) Starts with C2/CP2/VG

T0 Total: 5 checks

Tier 1: Metadata Extraction & Validation (T1)
# Field Extraction Method Pass Criteria
1 Contract Number Multi-pattern regex (3 patterns) Matches C2 or CP2/VG
2 Document Title Regex: ^(Title|Subject|Document Title) Length ≥ 5 chars
3 Revision Number Regex: ^(Revision|Rev\.?) Not draft marker
4 Issue Purpose Enum lookup IFU, IFI, IFC, IFR, IFT, etc.
5 Security Classification Enum lookup Company Use, Confidential, Restricted, Public
6 Document Type Enum lookup SOW, MDR, SDR, DWG, SPE, CAL, etc.
7 Discipline Code Enum lookup CMS, CIV, CME, ELE, INS, MEC, PIP, etc.
8 Originator/Contractor Regex: (From|Originator|Contractor) Non-empty, valid org name
9 Document Date Regex: Date pattern Valid date format
10 Supplier Name Regex: (Supplier|Vendor) Non-empty, valid name format
11 PO Number Regex: (PO|Purchase Order) Valid PO format
12 Doc Number Matches File Name String comparison Exact match after removing extension
13 Second Cover Page Regex: DOCUMENT CONTROL | DC Information DC block present
14 Contract Reference Validation Regex: ^C2[-\s] | ^(CP2|VG) Code matches C2 EPC-BOP
15 Correct Contract Submission Regex: CP2 | VG CP2 | Calcasieu | C2 Project = CP2

T1 Total: 15 checks
